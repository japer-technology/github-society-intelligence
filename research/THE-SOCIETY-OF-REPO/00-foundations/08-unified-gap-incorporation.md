## Implementation order alignment

The documentation layout supports the phased implementation order from the unified gap set:

1. Cognitive structure primitives and provenance
2. Scale, routing, and learning quality mechanisms
3. Reflective, constitutional, and ecology-level governance

## Notes on practical interpretation

This architecture uses governed technical approximations and does not claim sentience:
- soft inhibition is used instead of psychological claims about taboo/humor
- introspection records and blind-spot logs are used instead of claims of self-awareness
- workspace visibility plus explanation protocols are used instead of consciousness claims
- provenance and ideals artifacts are used instead of unverifiable oracle claims

## Verification checklist

Use this checklist when auditing future changes:
- every non-trivial settlement cites frame, evidence, method, confidence, and unknowns
- every durable artifact declares a representation class and typed links
- insulation map updates accompany new high-coupling interactions
- quarterly evolution includes credit assignment and dialogical quality review
- constitution changes follow stability-tier friction requirements
- research-crosswalk is updated when adding new cognitive mechanisms
