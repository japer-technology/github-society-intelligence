# Owner Briefings

The owner-briefings workspace is the society's interface to the owner.

Only the owner-briefing agency writes here. All other agencies produce proposals; owner-briefing synthesises and delivers them.

---

## Structure

```
owner-briefings/
  {date}/
    {briefing-id}.md
    {approval-request-id}.md
```

---

## Briefing types

| Type | Contents |
| --- | --- |
| `briefing` | Summary of completed analyses and settled actions |
| `approval_request` | Formal request for human approval before action proceeds |
| `action_confirmation` | Confirmation that an approved action was completed |
| `alert` | High-urgency item that needs immediate owner attention |

---

## Briefing format

```markdown
# Briefing — {date}

## Priority items
{list of items requiring action or decision}

## For your information
{list of completed items with outcomes}

## Approval required
{list of pending approvals, if any}

---
*Generated by owner-briefing | Settlement: {settlement-id} | {timestamp}*
```

---

## Owner response

The owner responds to approval requests by:
- Adding a specific confirmation comment to the linked issue
- Merging a PR (for constitutional/policy changes)
- Applying an `approved` or `denied` label

All responses are recorded in the relevant settlement record.
