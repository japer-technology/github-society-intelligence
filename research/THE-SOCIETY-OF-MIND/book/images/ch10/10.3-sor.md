# Figure 10-3 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [10-3.md](10-3.md) — *Piaget's water-jar experiment.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

Figure 10-3 is the second, isomorphic conservation experiment.
Different substrate (water rather than eggs), same architecture of
disagreement. This analysis records what the implementation does
with the idea that *one cognitive pattern recurs across domains*.

---

## The ideas Figure 10-3 actually carries

1. **A second instance of the same pattern.** Conservation reappears
   in a new medium; the pattern is not specific to eggs.
2. **Continuous quantity, not discrete count.** Liquid volume admits
   no integer answer; the cue *count* is unavailable.
3. **Two cues now compete (height, width) instead of one.** *Tall*
   and *thin* together produce a visual signal that pulls strongly
   in one direction.
4. **A new low-level competence — pouring.** The medium itself
   introduces an act (pouring) that has no analogue in 10-1/10-2.
5. **The history argument generalises.** *"It's the same water"* is
   structurally identical to *"they're the same eggs"*.
6. **The age-stratified verdict pattern is preserved.** The same
   architectural claim survives a change of substrate.

---

## What the implementation already absorbs

### 1 — Substrate-independent pipeline

The phase chain `guard → normalize → … → settle → act → remember`
([`02-workflow-design.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/02-workflow-design.md))
applies to any trigger event regardless of payload type. The plan
honours the idea that *the same cognitive pattern* runs across
different inputs.

### 2 — Persisted patterns across runs

The decisions memory under
`THE-SOCIETY-OF-REPO/06-memory/decisions/` and the per-run state
described in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
allow the same shape of judgement to be applied to multiple
artefacts over time.

### 3 — Domain-specific leaf agencies are first-class

[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
groups agencies by family (perception, memory, code, safety,
identity, integration, assembly, meta-admin). New leaves can be
added to a family without changing the upper structure, which
matches the figure's implicit claim that domain-specific cues plug
in under reusable administrators.

---

## What the implementation does not yet take into account

### A — No notion of "the same problem in a new domain"

The plan handles many runs but does not represent *"this run
instantiates the same cognitive pattern as that earlier run."* There
is no `pattern:` or `isomorphic_to:` field on a settlement. Piaget's
"the eggs experiment and the water-jar experiment are the same
experiment" claim has no operational counterpart.

### B — Multiple cues pulling in the same direction

Figure 10-3 adds a second visual cue (*thin*) that *reinforces* the
first (*tall*). The plan's critic signals are independent; nothing
in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
declares that two critics' agreement counts as stronger evidence
than either alone. Reinforcement between critics is not modelled.

### C — Continuous quantities

Critic signals carry a discrete `verdict` (the schemas in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md))
and a free-text summary. There is no canonical place to record a
continuous measurement (a magnitude, a ratio, a confidence) the way
*height* and *width* are continuous in the figure.

### D — Pouring: an action that *changes* the scene

Pouring water from one jar to another is an *act* the experimenter
performs, observed by the subject. The plan models acts via the
`act` phase
([`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md))
but only by the agent itself, not by an external operator. A
disturbance applied *to* the workspace *by a human or another
agency* is not a first-class event class.

## Implication for the plan (no changes proposed here)

Figure 10-3 is where the chapter says *the architecture, not the
substrate, is the explanation*. The implementation pipeline is
substrate-independent in the right way, but it does not yet label
pattern recurrence between runs, record reinforcement between
critics, or carry continuous magnitudes. Any move to add a
`pattern:` relation, a critic-agreement scoring rule, or a
continuous-magnitude field would touch
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
and
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
