# Figure 10-1 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [10-1.md](10-1.md) — *Piaget's eggs and cups, BEFORE.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

Figure 10-1 is a *baseline* panel: it shows the easy case before the
disturbance. This analysis records what the implementation already
does with baselines and what it does not yet do with the idea that a
baseline must be captured *as data* before any later judgement is
made against it.

---

## The ideas Figure 10-1 actually carries

1. **The baseline is a first-class observation.** The *before* panel
   exists because the diagnostic value of the *after* panel depends
   on a previously-recorded state.
2. **Agreement across age groups is itself evidence.** When all
   observers agree, the conflict that the rest of the chapter
   explains is provably absent here.
3. **One-to-one correspondence as a perceptual cue.** Pairing the
   rows lets a single low-level agent ("they line up") settle the
   question without recourse to counting or history.
4. **A controlled scene.** Five eggs, five cups, plain background:
   the figure isolates the variable that will change in 10-2.
5. **The verdict is captured as a quotation.** The caption records
   what subjects *say*, not what the experimenter infers — the data
   is the utterance.
6. **The figure presupposes a recording medium.** Whatever else
   Piaget does, he writes the verdicts down; the *before* state has
   to persist long enough to be compared with *after*.

---

## What the implementation already absorbs

### 1 — Baselines as captured state

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
gives every run a `state/<run-id>/` directory containing
`workspace.md`, `blackboard.md`, `signals.jsonl`, `report.md`. The
*before* of any settlement is therefore recoverable from the
preceding run's persisted state.

### 2 — Verdicts captured verbatim

[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)
treats critic and censor outputs as structured signals
(`signals.jsonl`) rather than as inferred summaries. The "what was
said" is preserved alongside the "what was concluded", matching the
figure's reliance on quoted verdicts.

### 3 — Controlled scenes via the perceive phase

The `perceive` phase
([`02-workflow-design.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/02-workflow-design.md))
normalises the trigger event into a stable scene before any agency
deliberates. This is the operational equivalent of presenting eggs
and cups in a known initial layout.

---

## What the implementation does not yet take into account

### A — A "no-conflict" outcome is not a recorded class

The figure's diagnostic point is that *agreement* is meaningful data.
The plan's signal vocabulary
([`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md))
records what critics *say*, but there is no first-class class for
"all critics agreed, therefore no deliberation was required." A run
in which every signal converges is logged as a normal run, not as a
baseline.

### B — The before/after pair has no schema

There is no manifest or memory shape that says *"this settlement
references that earlier settlement as its baseline."* Decisions
under `THE-SOCIETY-OF-REPO/06-memory/decisions/` are individually
addressable but not paired. Piaget's method — *the same scene,
disturbed* — is therefore not directly expressible.

### C — One-to-one correspondence has no analogue

Figure 10-1's cheap settling cue (pairing) corresponds to nothing in
the plan. There is no critic whose only competence is *"check that
two collections align element-for-element"*. The closest piece —
`critic.scope` — operates on diff extent, not on cardinal pairing.

## Implication for the plan (no changes proposed here)

Figure 10-1 is the figure that makes the next figure legible: without
a captured *before*, *after* is not a measurement. The implementation
plan persists state per run but does not yet treat *baselines* as a
named relation between runs, nor does it have a class for the
"everyone agreed" outcome. Any move to add such a relation — for
example, a `baseline_of:` field on a settlement, or a
`signal.kind: convergence` — would touch
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
and
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
