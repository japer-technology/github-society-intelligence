# Figure 10-4 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [10-4.md](10-4.md) — *Tall and Thin in isolation.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

Figure 10-4 makes the experimental point that the individual judges
(*Tall*, *Thin*) already exist and already fire *correctly* — the
trouble arises only when they meet. This analysis records what the
implementation does with the idea that conflict is the diagnostic,
not the bug.

---

## The ideas Figure 10-4 actually carries

1. **The constituent agents are independently competent.** Each
   produces the right verdict when it is the only relevant cue.
2. **Single-cue contexts are easy.** Cognitive trouble is specific to
   conflict between agents, not to either agent in isolation.
3. **A judge is identified by *what it attends to*.** *Tall* attends
   to height; *Thin* attends to width.
4. **Naming the agent precedes naming the conflict.** Before one can
   say "tall and thin disagree", each has to be a stable
   addressable unit.
5. **Test cases must isolate the agent under test.** Each vignette in
   the figure is a controlled environment for one agent.
6. **The verdict can be wrong even when the agent is right.** *"This
   one has more because it's taller"* is the correct local rule
   applied to the wrong global problem.

---

## What the implementation already absorbs

### 1 — Independent, addressable critics

[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
catalogues critics as stable, individually addressable units
(`critic.evidence`, `critic.scope`, `critic.consistency`,
`critic.taste`). Each has an explicit competence; each can be
invoked independently. This matches the figure's *Tall* and *Thin*
as named, freestanding judges.

### 2 — A judge is identified by what it attends to

Each critic manifest declares its scope (paths, signals, frames it
fires on). *What an agent attends to* is therefore a first-class
property of the agent, matching the figure's framing.

### 3 — Single-critic runs are well-defined

The pipeline does not require a quorum; a run in which only one
critic fires is a valid run. The "isolated agent" case from the
figure is operationally normal.

---

## What the implementation does not yet take into account

### A — There is no catalogued "single-critic test harness"

The figure's vignettes are *test cases* for *Tall* and *Thin*. The
plan describes the agency catalogue and the runtime pipeline but
does not include a per-critic test harness in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
or
[`10-bootstrap-checklist.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/10-bootstrap-checklist.md).
The idea *"show me a scene where this critic should fire and verify
that it does"* has no named operational shape.

### B — "Locally right, globally wrong" is not a signal class

The figure's most subtle claim is that the agent gives a *correct*
local answer to an *incorrect* framing of the question. The plan's
signal schemas in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
distinguish `verdict: pass/fail/abstain` but not `verdict:
locally-correct-globally-misapplied`. A critic that is right about
its local rule but wrong for the situation looks identical to a
critic that is right outright.

### C — Cue exclusivity is not represented

The vignettes work because *only* the relevant cue is present. The
plan does not have a way to say *"in this scene, only critic X
applies; suppress others"*. `inhibits:` exists at the agency level
in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
but is a relationship between agencies, not a scene-level exclusivity
declaration.

## Implication for the plan (no changes proposed here)

Figure 10-4 is reassuring for the plan: the existence of individually
competent, addressable critics is exactly what the implementation
already provides. The gap is in *evaluating* those critics — there
is no place to keep their single-cue test cases, and no way to
record that a critic was correctly applied to the wrong question.
Any move to add a per-critic test catalogue, a `locally-correct`
verdict class, or a scene-level cue-exclusivity declaration would
touch
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and
[`10-bootstrap-checklist.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/10-bootstrap-checklist.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
