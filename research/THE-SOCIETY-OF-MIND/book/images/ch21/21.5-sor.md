# Figure 21-5 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [21-5.md](21-5.md) — *Automatic finding machine.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 21-5 — the
APPLE-NEME line setting partial states in COLOR / SHAPE / SIZE so
that LOOK-FOR and MOVE-ARM-TO act *without* explicit instructions
from above — and checks each one against the implementation plan.

---

## The ideas Figure 21-5 actually carries

1. **A neme as a wide activator.** APPLE-NEME does not call anyone;
   it simultaneously biases COLOR, SHAPE, SIZE toward partial states
   that fit "apple."
2. **Action without instruction.** LOOK-FOR fires *because* its
   context already encodes red, round, apple-sized — not because
   something told it "look for an apple."
3. **Shared context is the addressing mechanism.** The link between
   APPLE-NEME, SEE, LOOK-FOR, PLACE, and MOVE-ARM-TO is not a
   message but a *common state* that they all read.
4. **A focus pronome (PLACE).** Once PLACE is filled, MOVE-ARM-TO
   fires for the same reason: its context is now consistent with a
   target.
5. **Feedback through MOVE EYES → SEE.** Acting changes what is
   perceived, which changes the context, which selects the next
   action. The figure is a loop, not a chain.
6. **Automatism as the absence of a controller.** The figure has no
   top-level "decide what to do next" node; the right thing happens
   because the shared partial states make only one continuation
   sensible.
7. **Polyneme → many partial meanings.** The same APPLE-NEME wakes
   multiple agencies into compatible partial states; that is the
   *poly* in *polyneme*.

## What the implementation already absorbs

### 1 — Polynemes as wide activators

[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
defines polynemes (path-, label-, phrase-) with `excite:` and
`inhibit:` maps that raise or lower weights on many agencies at once.
APPLE-NEME is the same kind of structure: one symbol, many
simultaneous biases.

### 2 — Activation without explicit invocation

[`02-workflow-design.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/02-workflow-design.md)
describes the `activate` phase as a competition: agencies whose
`activates_on` conditions match are *invited*, not *called*. This is
the operational analogue of LOOK-FOR firing "because the context is
right."

### 3 — A current-focus surface

[`11-mapping-sor-to-implementation.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/11-mapping-sor-to-implementation.md)
maps SOR's `current-focus/` to
`.forgejo-society/workspace/current-focus/current.md`. This is the
plan's closest analogue to PLACE — a single, well-known location
that other units read to know "what is being attended to right now."

### 4 — A shared workspace as the address

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
keeps `workspace.md` and `blackboard.md` as the human-readable face
of the per-run state. Agencies in the plan address each other less
by name and more by reading the same workspace — the same
mechanism Figure 21-5 calls "shared context."

## What the implementation does not yet take into account

### A — Polynemes set categorical states, not partial states

The plan's polynemes excite or inhibit *agencies* (weight 0..1 on a
named agent). They do not set *the state of a sub-agency* —
"COLOR is biased toward red, SHAPE toward round, SIZE toward
apple-sized." There is no concept of "this neme sets sub-agency X's
internal partial state."

### B — No COLOR / SHAPE / SIZE-style attribute agencies

Figure 21-5 presupposes small attribute agencies (COLOR, SHAPE,
SIZE) whose role is to *hold* a partial state set by an incoming
neme. The plan does not ship attribute agencies. The perception
family agencies in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
classify the *whole stimulus*, not individual attributes of an
object being attended to.

### C — Feedback loop is not present

The figure's MOVE EYES → SEE loop closes within a single act of
attention: acting on the world changes what is perceived. The plan's
phase pipeline runs once per stimulus, top to bottom; there is no
intra-run loop that returns to `perceive` after `act`. Re-perception
only happens on the next stimulus.

### D — No automatism without a controller

The plan's `deliberate` phase is essentially a controller: it
chooses which agency's output to act on. Figure 21-5's point is
that *no such controller is needed* once context is rich enough.
The plan does not have a "no-deliberation" mode where act simply
follows from shared state without an explicit selection step.

### E — PLACE / focus is read, not actively maintained

`current-focus/current.md` is a single file that some agency must
update. The figure's PLACE is *filled by upstream activity* —
LOOK-FOR's success is what fills PLACE, automatically, without an
explicit write. The plan has no agency whose job is "maintain
current-focus by side-effect of perception."

### F — A neme is matched against the stimulus, not against a persistent object representation

Polynemes match `paths`, `labels`, `phrases`. Figure 21-5's
APPLE-NEME refers to a persistent representation of *apple* — an
object that survives across phases and whose partial-state
imprint persists in COLOR / SHAPE / SIZE. The plan has no
persistent per-run object representation; what survives between
phases is text in the workspace, not a structured object-neme.

### G — Cross-modal automatism

The figure works because vision (SEE), spatial focus (PLACE), and
motor action (MOVE-ARM-TO) all read the same shared state. The
plan's agencies are all text-in / text-out and share the workspace
file, which is close in spirit — but the workspace is unstructured
prose, not a set of partial-state fields that distinct modality
agencies can each read narrowly.

## Implication for the plan (no changes proposed here)

Figure 21-5 is the figure where action *follows* from shared state
rather than from instruction. The plan honours wide activation
(polynemes) and a current-focus surface, but does not honour
attribute agencies, partial-state biases, the perception-action
feedback loop, or controller-free automatism.

The gaps above are recorded here as analysis, not as a change
request. Any move to close them — adding attribute agencies,
extending polynemes to set sub-agency partial states, allowing
re-perception within a run, or shipping a controller-free `act` path
— would touch the agency, frame, and polyneme schemas in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
and
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the workspace conventions in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
and the workflow design in
[`02-workflow-design.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/02-workflow-design.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
