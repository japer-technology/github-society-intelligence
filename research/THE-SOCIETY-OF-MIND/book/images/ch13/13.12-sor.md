# Figure 13-12 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [13-12.md](13-12.md) — *Program collapsing into
a script.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 13-12 and
checks each one against the implementation plan.

---

## The ideas Figure 13-12 actually carries

From the caption and §13.5 *Learning a script*:

1. **A program and its compiled script are two distinct
   artefacts.** Both produce the same drawing; one deliberates,
   the other does not.
2. **Compilation strips checks, not actions.** Every "consider
   next feature" line is removed; every "draw X" line is kept.
3. **The compiled script is shorter than the program.** Six
   lines collapse to five. Scripts are economical.
4. **The arrow from program to script is itself a process.**
   Compilation is a runtime event, not a manual rewrite.
5. **The compiled script no longer consults the agencies the
   program consulted.** Whatever decided "should I draw a body?"
   is not on the right-hand side.
6. **Decisions are baked into order.** The script's authority
   for drawing arms after the body comes from sequence, not from
   re-evaluation.
7. **The program is preserved alongside the script.** Both are
   visible in the figure; the script does not delete its
   precursor.
8. **Scripts are situation-specific.** This script draws *this*
   kind of figure; a new figure type would need a new script (or
   a new compilation pass).

## What the implementation already absorbs

### 1 — K-lines as prior-activation records

K-lines in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
preserve a useful past activation so it can be reactivated
later. This is a partial form of point 2 — the *outcome* of a
deliberation is retained without rerunning the deliberation.

### 2 — Manifests are preserved when their behaviour changes

Differentiation and self-modification keep the prior manifest
in git when a new one ships. Point 7 (compiled script preserves
its program) is honoured by version control.

### 3 — Catalogues are situation-specific

The first-ship catalogues in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
are written for the situations the plan expects to meet. Point
8 is consistent with how the plan grows.

## What the implementation does not yet take into account

### A — No script artefact distinct from the deliberating manifest

The plan's representation of a unit is its manifest, which is
always the *program*. There is no second artefact — a
`script.<name>` — that records "the compiled action sequence
for this manifest, when the situation matches the K-line." The
two boxes of Figure 13-12 collapse to one in the plan.

### B — Compilation is not a runtime event

Point 4 — that compilation is itself a process the runtime
performs — has no schema. Manifests are revised by humans (or
proposed by `agency.meta-admin.differentiation-broker` and
approved by humans). There is no `compile` phase, no
`agency.meta-admin.script-compiler`, and no settlement kind
`settlement.script-compiled`.

### C — Sequence as authority is not modelled

Point 6 — that the compiled script's authority comes from
order rather than from re-evaluation — has no representation.
Authority in the plan is granted by the authority-registry
levels `read | draft | propose | act | govern | human`; there
is no "order-authority" mode that says "this step runs without
re-asking because it sits at position N in a compiled
sequence."

### D — Stripped checks are still required

A script that bypasses `criticize` and `censor` would violate
the safety posture of
[`07-policies-and-safety.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/07-policies-and-safety.md).
The figure's "no consultations" property is incompatible with
the plan's current safety model unless scripts run with a
restricted authority that does not need those checks. The
restricted-authority idea does not exist either.

### E — No metric for "this could be a script"

The runtime could in principle observe "this deliberation has
produced the same action sequence N times in a row, so it is a
candidate for compilation." No such metric is collected
([`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
records signals and settlements but not action-sequence
recurrence).

### F — Situation-specific scope is not declarable

Point 8 — scripts are tied to a kind of situation — would need
a `applies_when:` field on the script artefact. With no script
artefact (gap A), the field is moot, but its absence means even
a hand-written shortcut has no way to say "use me only when
the situation matches frame X."

## Implication for the plan (no changes proposed here)

Figure 13-12 is the figure that earns the *script* as a unit
distinct from the program that produced it. The implementation
plan honours K-lines, which retain outcomes, and version
control, which retains precursors. It does not yet honour the
two-artefact picture — a deliberating program and a compiled
script — and so cannot represent the speed gain Figure 13-11's
epigraph promises.

The gaps above are recorded here as analysis, not as a change
request. Any move to close them — for example, a `script.<name>`
artefact in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
a `compile` phase in
[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md),
an `agency.meta-admin.script-compiler`, or an
order-authority mode in
[`07-policies-and-safety.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/07-policies-and-safety.md)
— would touch
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the authority registry in
`THE-SOCIETY-OF-REPO/01-governance/authority-registry.md`, and
the identity scopes in
`THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`, and so falls
under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
