# Figure 13-4 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [13-4.md](13-4.md) — *Erasing boundaries in
Tower-Arch.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 13-4 and checks
each one against the implementation plan.

---

## The ideas Figure 13-4 actually carries

From the caption and §13.2 *Boundaries*:

1. **The mind removes boundaries the world supplies.** Many real
   blocks are merged into two pillars and a lintel.
2. **Suppression is as much a perceptual act as supplementation.**
   The companion move to Figure 13-3: the same template that adds
   missing edges also hides real ones.
3. **The template selects which edges count.** Edges aligned with
   *body* or *support* survive; orthogonal seams between bricks
   are erased.
4. **Aggregation is the inverse of subdivision.** Figure 13-4
   collapses parts; Figure 13-3 splits a whole. Both serve the
   same uniframe.
5. **Loss of detail is intentional, not accidental.** The
   collapsed reading is preferred even when the brick-by-brick
   reading remains available.
6. **The same description can apply to objects with very
   different part-counts.** A two-block arch and a forty-block
   arch both read as *body + supports* under the template.

## What the implementation already absorbs

### 1 — Summary tiers collapse detail intentionally

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
and `agencies/assembly/*` describe summary tiers that fold many
signals into one brief. This is point 5 (loss of detail is
intentional) applied to outputs: the brief is preferred even
though the underlying signals survive in git.

### 2 — Variable-arity inputs to one description

Catalogue items in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
do not require a fixed number of input signals; an agency can
react to one signal or to many. Point 6 (same description over
different part-counts) is structurally available.

## What the implementation does not yet take into account

### A — Edge-suppression is not a perceptual primitive

`perceive` and the signal schemas in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
do not have an operation called "merge these N signals into one,
because the active frame does not care about the boundaries
between them." Aggregation in the plan happens at the *summary*
layer, not at the *perception* layer. Figure 13-4 is about
suppressing detail *before* the description is computed, not
after.

### B — Template-driven selection of edges has no representation

Point 3 — that the template chooses which edges survive — has
no operational analogue. Frames in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
can select which agencies fire; they cannot select which fields
of an incoming signal are visible to those agencies. The "fade"
of internal block edges is, in plan terms, not expressible.

### C — Aggregation and subdivision are not paired moves

The plan treats Figure 13-3's move (add boundaries) and Figure
13-4's move (remove boundaries) as unrelated. There is no shared
schema for "the template is currently re-drawing the part
structure" of which both supplementation and suppression are
instances. They would each need their own (currently absent)
machinery.

### D — Lossy descriptions are not labelled as such

When a summary tier collapses signals, the brief does not carry a
field saying "I have intentionally discarded the following
distinctions." Critics downstream cannot know which distinctions
were dropped on purpose and which were never present. Point 5
(intentional loss) is performed but not recorded.

### E — No "view under template T" addressable artefact

A maintainer cannot ask the workspace for "the current run as
read under `frame.regression-debug`" as a distinct object from
"the current run as read under `frame.self-modification`." The
run has one description in force. Figure 13-4 invites multiple
collapsed readings of the same scene; the plan supports one.

## Implication for the plan (no changes proposed here)

Figure 13-4 is the suppression-side companion of Figure 13-3. The
implementation plan honours collapsing at the summary layer, but
does not yet honour collapsing at the perception layer, nor record
which distinctions were dropped on purpose by the current template.

The gaps above are recorded here as analysis, not as a change
request. Any move to close them — for example, a frame-driven
field-filter in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
a "dropped distinctions" field on summary tier outputs, or paired
add/remove-boundary primitives in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
— would touch the manifest schema in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
and the identity scopes in
`THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`, and so falls
under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
