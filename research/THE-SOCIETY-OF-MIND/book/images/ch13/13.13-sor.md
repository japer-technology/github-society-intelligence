## The ideas Figure 13-13 actually carries

From the caption and §13.6 *The frontier effect*:

1. **An experiment can have a baseline that is uninteresting on
   its own.** The figure exists only to make the next figure
   meaningful.
2. **The system being studied performs well in the easy
   regime.** Small displacement, accurate drawing.
3. **Accuracy is a function of where the input sits in the
   space, not of the input's identity.** The same task fails
   later (Figure 13-14) under a different displacement.
4. **A "no-effect" baseline is information.** It says: the
   procedure is not always broken, so what does it depend on?
5. **Failure modes are local, not global.** The procedure is
   competent in one region of the input space and not in
   another.

## What the implementation already absorbs

### 1 — Critics run per situation

`critic.evidence`, `critic.scope`, and the other entries in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
evaluate each activation on its own merits. The plan does not
assume a unit is uniformly correct or uniformly broken; it
checks per run. Point 5 (failures are local) is honoured.

### 2 — Settlements record outcomes

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
preserves each run's settlement. The data needed to compare
"easy regime" and "hard regime" runs is on disk.

---

## What the implementation does not yet take into account

### A — No notion of an input "regime"

Point 3 — accuracy depends on *where* in the input space the
task sits — would need a way to characterise the situation
beyond its labels and paths. The plan tags inputs by what
triggered which frame, not by where they sit on any continuous
axis. No critic or memory entry says "this run came from the
near-edge region of input X."

### B — Baseline runs are not labelled as baselines

Point 4 — that a normal-success run is informative as a
contrast — is not represented. Every settlement is treated as
an end in itself; nothing in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
groups runs into "baseline" and "stress" sets.

### C — Per-region competence is not modelled

The plan's competence model is per-agency, not
per-agency-per-region. A unit is competent or not; there is no
field on a manifest that says "this unit is reliable in regime
A, suspect in regime B." Point 5 (local failure) cannot be
named precisely.

### D — Comparison across runs is manual

To establish "Figure 13-13 succeeds, Figure 13-14 fails,
therefore the procedure has a regime problem" requires
comparing two settlements. There is no aggregation layer above
settlements that performs this comparison; a human reads git.

### E — The "set up the next figure" role is not modelled

Figure 13-13's value is largely setup. The plan has no record
type for "this run was kept because it serves as a contrast for
that run." Useful pairs of contrasting runs are not first-class
in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md).

## Implication for the plan (no changes proposed here)

Figure 13-13 is the figure that earns the *baseline*. The
implementation plan honours per-run evaluation, but does not
yet honour input-regime characterisation or paired
contrast-runs that the upcoming Figure 13-14 will turn into a
finding.

The gaps above are recorded here as analysis, not as a change
request. Any move to close them — for example, an
`input_regime:` field on settlements, a `baseline`/`stress`
tag, or a `agency.meta-admin.regime-comparator` — would touch
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
and the identity scopes in
`THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`, and so falls
under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
