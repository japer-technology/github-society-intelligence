# Figure 19-4 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [19-4.md](19-4.md) — *Recognising apple by
conjunction.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This is the first introduction of *recognisers* as a distinct kind of
agent. The plan does not currently have recognisers as a kind.

---

## The ideas Figure 19-4 actually carries

1. **Recognition is a first-class agent role.** A unit whose job is
   to *fire when a configuration of inputs is present*.
2. **Conjunctive recognition.** The AND-agent fires only when *all*
   required features are simultaneously on.
3. **Features as named inputs.** SHAPE, COLOR, SIZE — each is a
   distinct wire from a distinct upstream agency.
4. **Naming the recognised object as the output.** The output of the
   AND is a single name (*"APPLE"*) — i.e. a polyneme by another
   name.
5. **Recognition is brittle by default.** If any one feature is
   missing, the AND does nothing.
6. **Recognition collapses many inputs into one symbol.** N input
   wires, one output wire.
7. **A recogniser is symmetric to a polyneme.** A polyneme broadcasts
   one symbol to many places; a recogniser collects many places into
   one symbol. The figure introduces the dual.

---

## What the implementation already absorbs

### 1 — Pattern-matching units exist

Critics ([`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md))
and censors are units that fire on patterns in proposals or in the
stimulus. They are the closest live analogue to a recogniser.

### 2 — Polynemes can be triggered by feature matches

Path, label, and phrase polynemes
([`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md))
fire when feature-shaped conditions are present in the stimulus. That
is an "AND on stimulus features" in spirit — the `match` block is
implicitly conjunctive across its fields.

## What the implementation does not yet take into account

### A — *Recogniser* is not a manifest kind

The manifest enum in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
lists agency, critic, censor, frame, polyneme, K-line, procedural,
and a few others. *Recognizer* is not among them. Anything that wants
to behave like an AND-agent today must masquerade as a critic, a
censor, or a polyneme-match.

### B — Recognition over *internal* states

A polyneme `match` block reads the *stimulus*. The figure's AND-agent
reads the current *internal* state of SHAPE, COLOR, SIZE — i.e. the
output of other agencies in this run. There is no schema for
"recogniser whose inputs are other agencies' current outputs."

### C — Recogniser output is not a polyneme

The figure's AND emits the symbol *APPLE*, which (by Figures 19-2 and
19-3) is itself a polyneme. The plan has no mechanism by which an
agency *emits* a polyneme. Polynemes are stimulus-triggered, not
agency-emitted (already noted in `19.2-sor.md` gap D).

### D — Strict conjunction is not expressible declaratively

The frame schema requires slots to be filled, which is an AND across
slots, but it does so over slow phase boundaries, not as a fast
recogniser. There is no compact `requires_all: [feature-1, feature-2,
feature-3] → emits: <polyneme>` shape.

### E — Recognisers do not appear in any phase

The pipeline phases are `guard → normalize → guardrail → perceive →
activate → deliberate → criticize → censor → settle → act → remember
→ report`
([`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)).
None of these is a *recognition pass*. The job of "scan the current
internal state and fire matching named recognisers" has no home.

## Implication for the plan (no changes proposed here)

Chapter 19 makes *recognisers* the partner of polynemes: polynemes
broadcast meaning, recognisers collect meaning. The plan has the
broadcast half and only an approximation of the collection half. The
two halves are clearly visible as a missing symmetry.

Adding recognisers as a manifest kind would touch the manifest enum
and authority registry in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
the activation protocol in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the pipeline phases in
[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md),
and the identity scopes in
`THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`, and so falls under
the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
