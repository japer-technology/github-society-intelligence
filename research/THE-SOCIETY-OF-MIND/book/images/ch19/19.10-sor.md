# Figure 19-10 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [19-10.md](19-10.md) — *Closing the ring of the
language-agency.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

The chapter's closing figure assembles every prior part into one
circular flow. This analysis asks how much of that ring the plan
already closes.

---

## The ideas Figure 19-10 actually carries

1. **A higher language-agency at the top.** A single named region
   that issues polynemes downward and receives recognisers upward.
2. **Descending nemes.** Words descend from the language-agency into
   the lower agencies as polyneme broadcasts.
3. **Memorizers between language and agencies.** A row of K-line
   banks intervenes on the descent path.
4. **A row of named lower agencies.** STRUCTURE, COLOR, TASTE, SHAPE,
   SUBSTANCE — a small, fixed cast of sensory or property agencies.
5. **Sensors as ground-level inputs.** SENSORS enter the row from
   below, separate from descending nemes.
6. **Ascending nemes from recognisers.** Recognisers (*fruity*,
   *round*, *apple-y*, *red*, *thin-peeled*) report upward into the
   cloud.
7. **The ring closes.** Three or four matching properties wake the
   *apple* polyneme, which descends and restores the rest. Reminding
   is the loop, not a separate faculty.

---

## What the implementation already absorbs

### 1 — A descending path exists

Polynemes in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
descend from stimulus features into the agencies they `excite`.
Stimulus → `perceive` → polyneme match → activation is the figure's
descending arrow at coarse grain.

### 2 — Memory can be queried mid-flow

`agency.memory.kline-retriever` ([`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md))
sits between activation and deliberation, which is roughly the
position the figure assigns to the memorizer row.

### 3 — A pipeline that returns to the user

`act → remember → report` closes a loop in the workflow sense:
output is produced, memory is updated, and the next stimulus can use
both
([`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)).
This is a *temporal* loop across runs, which is not the same as the
*intra-run* ring of the figure, but it is the closest live analogue.

### 4 — A central addressable language region, in spirit

`directive.*` polynemes and the `society <target>:` syntax make a
small region of the society addressable as if it were one place.
That is a partial implementation of the cloud at the top.

## What the implementation does not yet take into account

### A — No ascending neme path

The figure's right-hand arrow — recognisers reporting *upward* into
the language cloud — has no schema counterpart. Polynemes only
descend. There is no `ascending: true` flag, no `emits_neme:` field
on an agency, and no listener on the language side that consumes
upward nemes.

### B — Recognisers do not exist

Already recorded as `19.4-sor.md` gap A. Without recognisers, the
five upward labels in the figure (*fruity*, *round*, *apple-y*,
*red*, *thin-peeled*) have nothing to be the output of.

### C — The ring closure inside one run is absent

In the figure, three properties arouse *apple* and *apple* descends
to restore the others — within a single act of recognition. The
plan's loop is across runs (current run's `remember` → next run's
`perceive`), not within one run. There is no phase that says "if
recognisers fire mid-run, re-enter `activate` with the resulting
neme."

### D — Sensors are not a distinct input class

The figure separates *SENSORS* from *DESCENDING NEMES*. The plan has
one stimulus envelope. Tool outputs, file reads, and CI signals all
arrive at `perceive` through the same channel as user text. The
distinction "sensory ground vs descending word" is not modelled.

### E — The cast of lower agencies is fixed in the figure, open in the plan

STRUCTURE / COLOR / TASTE / SHAPE / SUBSTANCE is a closed list
chosen to make the apple loop concrete. The plan's families are open
and project-specific. That is healthier in general but means the
figure's specific loop has no specific home; nothing in the plan
says "this is our equivalent of the apple-loop."

### F — Reminding is not an architectural feature

Figure 19-10's headline payoff is that *reminding falls out of the
ring*. The plan supports remembering (K-lines, settlements, decision
records) but not reminding *as a wiring property*. If a partial
match against past state should automatically wake the full state, no
schema currently arranges it; an LLM step would have to do it
explicitly.

## Implication for the plan (no changes proposed here)

Figure 19-10 is the chapter's "complete machine": polyneme down,
recogniser up, K-line in the middle, ring closed, reminding emergent.
The plan supplies roughly the left half — descent, activation,
central memory — and almost none of the right half — recognisers,
upward nemes, intra-run closure. The asymmetry is real and is the
single most concise way to describe the gap between the chapter and
the implementation.

Closing the ring would touch the polyneme schema and the activation
protocol in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the manifest kinds in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
the pipeline phases in
[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md),
the K-line storage layout in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
the handoff/signal schemas in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and the identity scopes in
`THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`, and so falls under
the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
