# Figure 20-7 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [20-7.md](20-7.md) — *a network of branching
evidence-weighing agents.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 20-7 and checks each
one against the implementation plan. The aim is to mark, plainly, what
the plan already absorbs and what it does not yet name.

---

## The ideas Figure 20-7 actually carries

Pulled from the figure caption and Minsky's surrounding text in §20.9
*distributed memory*:

1. **The three-layer separation collapses on recursion.** Apply the
   transmitter / connector / receiver split inside itself and the
   layers merge into a single mesh.
2. **Every agent is simultaneously transmitter and receiver.** Roles
   are circumstantial, not assigned.
3. **Each agent is an evidence-weigher with a threshold.** It
   integrates inputs and fires when its threshold is crossed.
4. **Branching at both input and output.** Multi-in, multi-out is
   the default shape.
5. **A central band of *micronemes*.** Groups of nearby connection
   lines acquire local meaning and become first-class.
6. **Connectivity is not random.** Spatial / topological proximity
   matters; nearby connections recurrently form useful subunits.
7. **Subunits emerge from traffic, not from declaration.** What
   counts as a *microneme* is read off the network's behaviour, not
   pre-listed.
8. **The mesh is the substrate of distributed memory.** No single
   agent holds a memory; the pattern across agents is the memory.

These eight items are what the rest of this file tests against the
implementation plan.

## What the implementation already absorbs

### 1 — Every agency can emit and receive

[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
and
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
treat agencies as both signal sources and signal sinks. There is no
schema distinction between "transmitter agencies" and "receiver
agencies."

### 2 — Pattern-based activation

`activates_on:` lets an agency fire on combinations of inputs, which
is the evidence-weighing shape Figure 20-7 depicts.

### 3 — Branching topology

Many-to-many fan-out and fan-in are the pipeline's default
([`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)).
The mesh quality of Figure 20-7 is structurally compatible with the
plan.

### 4 — Distributed memory as committed state

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
keeps state in K-lines, settlements, and Git objects rather than in a
single agency. The "memory is the pattern, not a unit" idea has an
operational analogue: memory is the pattern of K-lines and
settlements that a frame reactivates.

### 5 — Polynemes as candidate micronemes

[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
defines polynemes — multi-target activation messages that wake
several agencies at once. A frequently-used polyneme is functionally
close to a microneme: a recurring bundle that has acquired its own
significance.

## What the implementation does not yet take into account

### A — Thresholds and weights are not first-class

Figure 20-7's agents have *thresholds*. The plan's `activates_on:`
clauses are pattern matches, not weighted sums. There is no
documented `threshold:` field, no per-input `weight:`, and no
notion of an agency "almost firing."

### B — Micronemes have no schema slot

A polyneme is a declared object. A *microneme* in Figure 20-7's
sense is an *emergent* bundle of co-active signals that has earned
the right to be treated as a unit. The plan does not specify a
mechanism for promoting recurring co-activation patterns into named
polynemes; the catalogue is hand-curated.

### C — No co-activation tracking

The plan keeps signal logs and settlements; it does not specify a
co-activation index ("these two signals fire together N% of the
time") that would be the raw material for microneme discovery.

### D — Topology / proximity is undefined

Figure 20-7 assumes that *nearby* connections matter — that the
network has a spatial or topological structure. The plan's agency
graph is implicit in `activates_on:` clauses and has no documented
notion of neighbourhood. "Nearby" is not addressable.

### E — Subunit promotion is not a policy

The plan does not name a meta-admin agency whose job is to detect
recurrent bundles and propose them as new polynemes. The closest
analogue is `agency.meta-admin.differentiation-broker`
([`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md))
which differentiates *agencies*, not connection patterns.

### F — Distributed memory is not explicitly named as such

The state model has the *ingredients* of distributed memory (K-lines,
polynemes, settlements stored as Git objects) but the documents do
not name "memory is the pattern across units" as a design invariant.
A single agency could, in principle, be designed as a centralised
memory without violating any current rule.

### G — Recursion of the three-layer model is not represented

Figure 20-7 is what happens when Figure 20-6's three layers are
re-applied inside themselves. The plan currently does not even have
the three layers cleanly separated (see [20.6-sor.md](20.6-sor.md)
gap A), so the recursive collapse cannot be discussed in plan terms
at all.

## Implication for the plan (no changes proposed here)

Figure 20-7 is the figure where memory stops being a *place* and
becomes a *pattern*. The implementation plan has the building blocks
— K-lines, polynemes, settlements, Git objects — but does not yet
specify thresholds, co-activation tracking, emergent microneme
promotion, or topology, and so cannot describe the pattern-as-memory
view explicitly.

The gaps above are recorded as analysis. Any move to close them — a
`threshold:` / `weight:` field on `activates_on:`, a co-activation
index in the state model, an `agency.meta-admin.microneme-broker`,
or a documented notion of neighbourhood in the activation graph —
would touch the manifest schema in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
the frames protocol in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the policies in
[`07-policies-and-safety.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/07-policies-and-safety.md),
the state model in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
the handoff schemas in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and the identity scopes in `THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`,
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
