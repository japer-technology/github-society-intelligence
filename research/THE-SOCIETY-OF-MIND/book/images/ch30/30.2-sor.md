# Figure 30-2 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [30-2.md](30-2.md) — *Mary's brain holds Mary's
model of the world.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

The figure draws one containment relation: the *world model* lives
inside the brain. The analysis below treats that relation, and the
absence of a "the world itself" node, as testable claims about the
implementation plan.

---

## The ideas Figure 30-2 actually carries

1. **A world model is a thing inside the agent.** It is not the
   world; it is structures the agent's own agencies can query.
2. **A boundary exists between agent and world.** The outer frame is
   what the brain is not.
3. **The model is bounded.** It fits inside the brain; it is not
   coextensive with the world.
4. **The model is *for being queried*.** It is defined functionally
   — as what other agencies can use to answer questions.
5. **There is no node labelled *world*.** The agent has no slot that
   represents the world itself, only models of pieces of it.
6. **The model is a single shape.** No internal partition is shown
   yet — sub-structure arrives in [30-3.md](30-3.md).

---

## What the implementation already absorbs

### 1 — A bounded internal substrate the run can query

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
defines a workspace and a memory layout that the runtime owns: a
finite tree under `.forgejo-society/` plus per-run scratch. Agencies
query this substrate via signals and frames
([`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)).
That is the operational form of "structures inside the agent that
other agencies can use to answer questions."

### 2 — The boundary between agent and world is enforced

[`07-policies-and-safety.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/07-policies-and-safety.md)
makes the workspace edge a hard one: censors (e.g. `censor.cloud-egress`,
`censor.fs-write`) intercept anything that would cross it. The frame
in Figure 30-2 is, in the plan, the union of those censors.

### 3 — Memory is functionally defined

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
describes K-lines, summaries, and settlements not as "what is true"
but as "what later runs can retrieve and act on." This matches the
figure's functional definition of a model: a structure judged by
whether queries against it succeed.

### 4 — No claim to represent "the world"

The plan never declares an agency, signal, or memory slot whose job
is to hold *the world*. Everything is scoped to a repository, a run,
a workspace, a signal stream. The absence of a "world" node in
Figure 30-2 is, by accident or design, mirrored in the plan.

## What the implementation does not yet take into account

### A — "World model" is not a named layer

The plan distinguishes *workspace* (current run), *memory* (across
runs), and *settlement* (a decided outcome)
([`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)).
It does not name an intermediate *world-model* layer — a stable,
queryable picture of the repository, its history, its collaborators,
its conventions. K-lines and summaries are pieces of one; nothing in
the plan composes them into a single addressable structure with the
shape Figure 30-2 draws.

### B — No "outside the boundary" type

Figure 30-2's frame implies a typed distinction between *inside the
agent* (the model) and *outside* (the world). Censors enforce the
boundary at runtime, but the schema in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
does not tag signals with their *origin side*. A signal sourced from
the live repository and one sourced from a stored summary share the
same shape. Mary's diagram presupposes that "where the structure
came from" is part of its identity.

### C — The model has no own identifier

In the figure, "Mary's model of the world" is the second node — it
has its own boundary and its own name. In the plan, the equivalent
collection has no id: there is no `memory.world-model.<repo>` or
`workspace.model.<run>` entity in
[`11-mapping-sor-to-implementation.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/11-mapping-sor-to-implementation.md).
The identity scopes in `THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`
include `memory` and `workspace` but not a composed-model scope.

### D — Queries against the model are not typed as queries

Activation in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
flows by signals and polynemes; retrieval against memory is one
agency among others (`kline-retriever`). The figure treats *querying
the model* as the model's defining operation. In the plan it is one
specialist among many, with no schema-level distinction between "ask
the model" and "do work in the world."

### E — The model cannot say "I do not contain that"

A central implication of the figure — the model is bounded, so some
questions fall outside it — has no operational analogue. There is no
field in signal or settlement schemas
([`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md))
for "this question was outside the model's scope"; an empty retrieval
is currently indistinguishable from a retrieval against an absent
fact.

## Implication for the plan (no changes proposed here)

Figure 30-2 is the *containment* figure: the agent has a model, the
model has a boundary, the world is outside it. The implementation
plan owns the boundary (via censors) and the substrate (via memory
and workspace), but does not name a composed *world-model* layer or
give it an identifier. Closing that gap would touch
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
the identity scopes in `THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`,
and the signal schema in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
