# Figure 14-7 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [14-7.md](14-7.md) — *Four things that are all "squares".*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

---

## The ideas Figure 14-7 actually carries

1. **One concept, several incompatible evidence shapes.** Corners
   only, edges only, brackets only, dot-lattice — all read as
   *square*.
2. **Recognition tolerates absent features.** Each panel is missing
   what one of the others has.
3. **Local evidence is sufficient but not unique.** Several
   different local patterns license the same global label.
4. **Completion is automatic.** The viewer supplies the missing
   sides, corners, or interior without effort.
5. **A concept is the union of its evidence patterns.** *Square* is
   defined by what suffices to evoke it, not by a single criterion.
6. **Redundancy enables robustness.** If one cue is destroyed,
   others still identify the form.

---

## What the implementation already absorbs

### 1 — Multiple polynemes can excite the same agency

[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
makes polynemes per-axis: path-polynemes, label-polynemes,
phrase-polynemes. Several of them may name the same agency in
their `excite:` map, so distinct evidence shapes (a path glob, a
label, a phrase) can all wake the same unit.

### 2 — Frames match on disjunctions

A frame's `matches` block is
`any_signals / any_labels / any_paths / any_phrases`. Any one of
those is enough to bias selection. The figure's "any of these cues
is enough" pattern is honoured at the frame-selection layer.

### 3 — Evidence is an explicit list

`Signal.evidence` in
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
is an array of typed items (`path`, `label`, `phrase`, `symbol`,
`diff`, `metric`, …). The schema is set up to record several
heterogeneous cues for a single signal, which is the *square*
situation.

---

## What the implementation does not yet take into account

### A — No redundancy-aware confidence

A signal carries `energy ∈ [0,1]` but the formula by which energy
relates to *number of independent evidence kinds* is not
specified. Figure 14-7's payoff — robustness comes from
*independence*, not from accumulation — has no operational form.
Two pieces of evidence of the same kind should count less than
two of different kinds; the schema does not say so.

### B — No completion / filling-in agency

The figure's viewer *adds* the missing sides. No first-ship agency
in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
is tasked with proposing the parts of a percept that *aren't there
yet but would complete it*. Ambiguity-detector flags the gap;
nothing fills it.

### C — Cue-loss tolerance is not measured

The figure asks: if one cue is removed, do the others still
identify the form? The plan has no test, no critic, and no
reinforcement signal that scores *how degraded* the evidence can
become before an agency stops activating. K-line reinforcement
([`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md))
tracks success/failure but not robustness to cue removal.

### D — A concept is not represented as a set of equivalence
classes

The figure presents *square* as four equivalence classes of
evidence. The plan stores K-lines and frames, but a K-line is a
single past activation pattern, not "here are the four substitutable
ways to evoke me." A concept-as-union-of-evidence-patterns object
has no schema.

## Implication for the plan (no changes proposed here)

Figure 14-7 asks the runtime to be robust to which cues are
present. The plan is set up to *accept* disjunctive cues but does
not yet *reward* independence or *complete* missing ones. The
gaps above are recorded as analysis, not as a change request. Any
move to close them — for example, an *evidence-independence* term
in signal energy, an `agency.perception.completion`, or a
`concept.schema.json` carrying equivalence classes of cues — would
touch the signal, polyneme, and memory schemas, and so falls under
the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
