# Figure 14-1 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [14-1.md](14-1.md) — *Body and support across many things.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 14-1 and checks each
one against the implementation plan. The aim is to mark, plainly, what
the plan already absorbs and what it does not yet name.

---

## The ideas Figure 14-1 actually carries

1. **A single distinction generalises across unrelated objects.** The
   same horizontal cut separates tripod, umbrella, car, tree, and
   table.
2. **The cut is functional, not geometric.** Nothing in the shapes is
   shared; what is shared is the role split *instrument above /
   servant below*.
3. **A shared reference plane.** The ground line lets the cut land in
   the same place across very different bodies.
4. **Reformulation by transport.** The figure's whole move is to
   *carry* a familiar distinction into a new realm and see whether it
   still cuts cleanly.
5. **The labels do the work.** Once `BODY` and `SUPPORT` are written
   in the margin, the eye reads the same partition into every object.
6. **Cross-realm vocabulary as a first-class artefact.** The labels
   are themselves the thing being reused, not the geometry beneath
   them.

## What the implementation already absorbs

### 1 — Family vocabulary that travels

[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
defines a small fixed set of agency families (`perception`, `memory`,
`code`, `safety`, `identity`, `integration`, `assembly`,
`meta-admin`). Every manifest carries an `agency:` field with one of
these values. That field is a cross-domain cut: the same word groups
units that share *role*, not *content*.

### 2 — Analogies as durable cross-realm records

The `memory/analogies/` tree (referenced by
[`11-mapping-sor-to-implementation.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/11-mapping-sor-to-implementation.md))
is the storage location for exactly this kind of transported
distinction. When a cut works in a new realm, that observation has a
named home.

### 3 — A shared "reference plane" for runs

`state/runs/<run_id>/stimulus.json`
([`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md))
is the ground line of any run: a single normalised event against
which every later artefact is positioned.

## What the implementation does not yet take into account

### A — No first-class "cut" or "distinction" object

A figure-14-1-style distinction (BODY/SUPPORT, instrument/servant) is
the kind of object the plan would want to name, store, and reuse. The
schemas in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
and
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
have manifests, frames, polynemes, K-lines — but no `distinction:` or
`dyad:` kind. A reformulation cannot be recorded as "we applied the
*X-over-Y* cut to a new domain."

### B — Cross-realm reuse is implicit

The `memory/analogies/` tree exists in the mapping table but has no
schema in this folder. There is no documented record shape for "this
cut from domain A also holds in domain B," and no critic whose job
is to *propose* such transports.

### C — No labelling agency

The figure's force comes from naming the regions. The plan has
narrators (`agencies/assembly/*`) that summarise; it does not have a
unit whose job is to *attach a reusable label to a region of state*
so that the same label can later activate the same cut elsewhere.

### D — Polynemes carry symbols, not distinctions

[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
defines polynemes as single symbols that excite many agencies. A
*distinction* — a paired top/bottom that must be applied together —
has no analogue. Polynemes wake things; they do not partition them.

## Implication for the plan (no changes proposed here)

Figure 14-1's content is a *reformulation primitive*: the act of
transporting a distinction across realms. The plan honours the
*outcome* of such reformulations (families, analogies as a memory
tree) but does not yet honour the *primitive itself* (a recordable,
schema-validated "cut" that can be carried from one domain to
another).

The gaps above are recorded as analysis, not as a change request. Any
move to close them — for example, a `distinction.schema.json`, a
`memory/analogies/*.analogy.yml` shape, or an
`agency.meta-admin.cross-realm-cut-proposer` — would touch the
manifest and memory schemas and the identity scopes, and so falls
under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
