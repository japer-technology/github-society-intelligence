# Figure 16-10 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [16-10.md](16-10.md) — *Stimulus versus simulus
across the vision levels.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas Figure 16-10 carries and checks each
one against the implementation plan.

---

## The ideas Figure 16-10 actually carries

1. **Perception is layered.** Retina → boundaries → regions → frames
   → relationships → goals.
2. **A stimulus enters at the bottom.** Real input climbs the stack.
3. **A simulus enters partway up.** An internal signal — typically a
   K-line — injects directly into a mid- or high-level agent.
4. **The cognitive effect is the same.** Downstream agents cannot
   tell the difference.
5. **Imagination is cheaper than perception.** It bypasses the lower
   layers.
6. **Imagined scenes lack detail but keep significance.** Bottom
   layers are absent; high-level meaning is intact.
7. **K-lines are the simulus carriers.** Memory can replay
   high-level states without re-running the senses.
8. **The same machinery serves perception and fantasy.** No separate
   "imagination subsystem" — the upper levels are dual-use.

## What the implementation already absorbs

### Perception is staged

The phase chain in [`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)
splits intake into `guard → normalize → guardrail → perceive →
activate → deliberate`. There is real layering, even if it is fewer
layers than Figure 16-10's six.

### K-lines exist as a first-class kind

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),

with `id: kline.<class>.<date>.<stimulus_id>`. The carrier from idea
7 is present.

### Branches as imagination

`society/<stimulus_id>/candidate-<n>` branches are documented in
[`02-workflow-design.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/02-workflow-design.md),
[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md),

### Frames as mid-level recognisers

`code-change`, `security-sensitive`, `self-modification`, `novel`

## What the implementation does not yet take into account

### A — Perception layers are not addressable

Figure 16-10's claim depends on being able to *inject at a specific
level*. The plan's `perceive` phase is a single stage; there are no
named sub-layers (e.g. `perception/raw`, `perception/boundary`,
`perception/frame`, `perception/goal`) that a K-line could target.
"Replay this K-line at the frame level, not the raw-stimulus level"
cannot be said.

### B — K-line replay is recall, not injection

Today, `kline-retriever` *fetches* relevant K-lines into the
deliberation context. Figure 16-10's simulus is stronger: the K-line
becomes the same kind of signal an external stimulus would be, and
travels through the same downstream machinery. There is no
"replay-as-stimulus" mode in the schemas.

### C — Branches-as-imagination is at the act layer

The plan's imagination lives at the *output* end (candidate
branches). Figure 16-10 puts imagination in *perception* — the same
machinery that sees real cups can be made to see imagined ones.
There is no perceptual-imagination analogue of `act`'s branches.

### D — No "this perception was a simulus" provenance

When a frame fires, the plan records *that* it fired, not *whether*
it fired from external stimulus or from a replayed K-line. A critic
auditing why an agency reacted cannot ask "was the trigger real or
remembered?"

### E — No cost model that prefers simulus

Idea 5 — imagination is cheaper — has no operational reflection.
Branches-as-imagination cost the same as real writes (same runner
time, same disk). There is no policy saying "prefer to test the
hypothesis on a recalled K-line before re-perceiving the world."

### F — Dual-use is not documented as such

The plan does not say "the same agencies serve both real input and
recall." It is implicitly true (a frame is a frame), but Figure
16-10's argument depends on that property being designed-for and
named, so it can be reasoned about.

## Implication for the plan (no changes proposed here)

Figure 16-10 names the cognitive value of imagination: the upper
levels of perception can run on memory alone. The implementation
plan has the *parts* — staged perception, K-lines, branches-as-
imagination — but does not yet wire them into the figure's dynamic
(addressable layers, replay-as-stimulus, simulus provenance,
imagination cost preference, dual-use as a stated design property).
Any of these would touch the runtime pipeline in
[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md),
the activation protocol in [`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the memory layout in [`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
and the handoff schemas in [`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
