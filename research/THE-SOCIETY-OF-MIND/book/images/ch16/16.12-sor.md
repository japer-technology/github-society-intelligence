# Figure 16-12 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [16-12.md](16-12.md) — *Infant proto-specialists in
cross-exclusion.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas Figure 16-12 carries and checks each
one against the implementation plan.

---

## The ideas Figure 16-12 actually carries

1. **Mood is the *visible* winner of an internal contest.** Many
   proto-specialists run in parallel; one controls the surface.
2. **Cross-exclusion produces sharp transitions.** No fade between
   moods, only switches.
3. **Expression is a scarce shared organ.** Voice, face, posture can
   only signal one state at a time.
4. **Clarity is itself adaptive.** A caretaker can only react to an
   unambiguous signal; the noncompromise rule serves communication.
5. **Internal pluralism, external monism.** The inside stays a
   society; the outside is forced to look like an individual.
6. **The set of proto-specialists is small and named.** *Hunger*,
   *Sleepiness*, *Affection*, *Play*, *Contentment*, *Hunger-Rage*
   — a finite roster, not a continuum.
7. **The same wiring as Figure 16-6 applied to emotion.** Emotion is
   not a new mechanism, just cross-exclusion at the mood layer.
8. **The other specialists keep working.** They lose expression, not
   activity.

## What the implementation already absorbs

### One conscious presenter

`agency.assembly.conscious-presenter` ([`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
[`00-overview.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/00-overview.md))
is the single surface output of a run. Many agencies deliberate;
exactly one piece of prose reaches the user. That is the
"internal pluralism, external monism" of idea 5 at the run scale.

### One settlement per stimulus

[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md):
"One settlement per stimulus." The plan picks one outcome, not a
blend.

### Summary tiers as the assembly layer

`agency.assembly.summary-tier-1`, `summary-tier-2`, … collapse many
parallel signals into the single voice that reaches the surface.
This is the structural analogue of mood-as-winner.

### Single conscious channel

[`02-workflow-design.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/02-workflow-design.md)'s
`report` phase emits one final user-facing comment per run — one
face, one voice.

## What the implementation does not yet take into account

### A — No "mood" or persistent affective state

Figure 16-12 is a story about a *state* that persists across moments
and then flips. The plan has no `state/mood.yml`, no
`memory/emotion/`, no schema where a long-running affective
"posture" of the society could live. Every run starts from neutral.

### B — Expression is not modelled as a scarce organ

The conscious presenter is *one*, but it is not declared a scarce
shared organ that competing agencies fight over. There is no
`organs/expression.yml` and no concept of contention specifically
*for the channel to the user*. The single-voice property is an
emergent fact of pipeline shape, not a documented constraint.

### C — Cross-exclusion is not used on assembly

As noted in [16.6-sor.md](16.6-sor.md) gap D, `inhibit:` exists but
no first-ship catalogue uses it. The assembly family in particular
does not have a mutual-inhibition wiring among summary tiers or
among candidate presenters — the choice of voice happens by
sequential tier collapse, not by competition.

### D — No "other specialists keep working" record

Idea 8 — the losers stay active, only the expression is suppressed
— would, in the plan, look like "non-winning agencies still produce
artefacts that are kept somewhere off-channel." Today, non-selected
candidate branches and unaccepted settlements are kept by git, but
there is no explicit "background activity log" that surfaces *which
specialists were running and what they thought*, separate from what
was reported.

### E — Sharpness of transition is not designed-for

The figure's sharp on-off switching has no documented analogue. The
plan does not say whether transitions between successive runs'
tones should be sharp or smoothed, because the plan does not yet
have a notion of *tone across runs* to switch (gap A).

### F — Adaptive clarity is not stated as a value

The figure argues that ambiguity is bad *for the receiver*. The
plan's report phase and conscious presenter produce one answer per
run, but the documentation does not justify this in terms of the
human reader's need for an unambiguous signal — it is stated as
plumbing, not as design ethics.

## Implication for the plan (no changes proposed here)

Figure 16-12 maps the chapter's arbitration mechanism onto
*emotion*. The implementation plan has the surface property — one
voice per run — but has not yet built the supporting structures:
persistent affective state, expression as a declared scarce organ,
mutual inhibition in assembly, a background-activity record of the
specialists that did not win, and an explicit design value of
clarity-for-the-receiver. Any of these would touch the manifest
schema in [`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
the memory layout in [`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
the runtime pipeline in [`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md),
and the activation protocol in [`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
