# Figure 16-9 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [16-9.md](16-9.md) — *Thirst exploiting See, Find,
and Get.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas Figure 16-9 carries and checks each
one against the implementation plan.

---

## The ideas Figure 16-9 actually carries

1. **A small agent achieves a large goal without owning the
   knowledge.** Thirst lacks a model of cups but still gets a cup.
2. **Exploitation, not invocation by argument.** Thirst sends
   *control*, not *data*: it arouses See's existing machinery rather
   than describing a cup.
3. **Side-arousal of high-level recognisers.** Thirst makes See
   hallucinate a cup at its highest levels — biasing perception, not
   bypassing it.
4. **Chains, not trees.** Find passes to Get; the agents form a
   horizontal pipeline, not a hierarchy.
5. **Each step retains its own logic.** Find decides for itself how
   to find; Get decides for itself how to grasp.
6. **Brevity is structural, not stylistic.** The short signal works
   only because the receivers already carry the long meaning.
7. **The same trick prefigures language.** *Please pass the cup* is
   exploitation by sound rather than by neural link.
8. **The exploiter need not know how the exploited works.** Thirst
   has no model of See, Find, or Get; the call still succeeds.

## What the implementation already absorbs

### Polynemes are the exploitation primitive

[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md)
defines polynemes as "symbol-like activators that wake many partial
meanings across agencies." That is exactly Figure 16-9's mechanism: a
short signal that triggers receivers whose internal logic does the
real work.

### Direct addressing exists

`directive.spock`, `directive.safety`, `directive.memory`,
`directive.critic` (phrase-polynemes in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md))
let one agent — or a human — name another by id and wake it. This is
Thirst-to-See lifted to the prose level.

### Chains across families

Cross-family activation is permitted: an `agency.code.patch-imaginer`
proposal can wake `critic.evidence`, which can wake `censor.*`, which
can wake `assembly/summary-tier-1`. Each link retains its own logic,
matching the horizontal Find → Get chain.

### Exploiter need not understand exploited

The polyneme system is content-blind. `directive.memory` does not
need to know how `kline-retriever` walks the memory tree; it only
needs the id. Idea 8 is in the plan's grain.

## What the implementation does not yet take into account

### A — Side-arousal of recognisers is not modelled

Figure 16-9's trick is that Thirst makes See *hallucinate* a cup at
its highest level. The plan's perception is driven from stimulus
content downward, not from drive content upward. No agency can
*bias* perception toward a category — there is no field like:

```yaml
arouses_recognition:
  category: <symbol-id>
  level: <perception-tier>
```

This is the operational gap behind everything Figure 16-9 implies
about top-down attention.

### B — No control-only signal

Polynemes carry an excitation weight and (sometimes) a frame
default; they do not carry "do this for me." A call-like primitive
— "Thirst asks See to look for cups, full stop" — has no schema. The
closest thing is direct addressing in prose, which is interpreted
freely by the recipient rather than executed.

### C — No declared chain template

Figure 16-9's See → Find → Get is a *standing chain* that any drive
can invoke. The plan has no `chains/<chain-id>.yml` registry; chains
emerge per run from polyneme excitation. A reusable
"see-find-get" capability cannot be declared once and called many
times.

### D — Exploitation is not separated from collaboration

The plan has *one* call mechanism (polynemes / direct addressing)
used for everything from "please remember this" to "please review
this diff." Figure 16-9 distinguishes *exploitation* (use another's
machinery as your own) from ordinary cooperation. The implementation
does not draw that line, so there is no place to apply different
audit, budget, or authority rules to exploitation specifically.

### E — Language-as-exploitation is not surfaced

Idea 7 — that natural language between humans and the society is
itself the same trick — is not recorded as a design property. The
`society <target>:` syntax exists, but the documentation does not
say "this is the externalised form of the polyneme mechanism." A
critic reasoning about prompt-injection cannot lean on that framing
because it is not written.

### F — No protection against parasitic exploitation

If Thirst can arouse See, so can anything else. The plan permits any
agency to wake any other via `excite:` weights and direct
addressing; there is no `callable_by:` allow-list on agencies, and
no per-link rate limit. Figure 16-9 assumes goodwill among
specialists; production cannot.

## Implication for the plan (no changes proposed here)

Figure 16-9 is the figure that justifies polynemes — and the
implementation plan takes the justification. What the figure asks
for that the plan does not yet provide is *top-down arousal*, a
*call-only* primitive distinct from prose addressing, *declared
chains*, and *callability allow-lists*. Adding any of these would
touch the activation protocol in [`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the manifest schema in [`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
the handoff schemas in [`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md),
and the safety policies in [`07-policies-and-safety.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/07-policies-and-safety.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
