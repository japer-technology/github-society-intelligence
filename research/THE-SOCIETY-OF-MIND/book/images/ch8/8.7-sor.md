# Figure 8-7 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [8-7.md](8-7.md) — *Two ways to wire a new memory.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 8-7 and checks each
one against the implementation plan.

---

## The ideas Figure 8-7 actually carries

1. **A memory built from raw agents grows linearly with detail.**
2. **A memory built from prior K-lines grows by composition.** Three
   pointers replace a dozen attachments.
3. **The composed form inherits the structure of its sub-K-lines.**
   What JACK, FLY, and KITE already know comes along for free.
4. **Composition is the path to economy *and* to abstraction.**
5. **Which prior K-lines to use is itself a cognitive step.** The
   composing agent had to recognise JACK, FLY, KITE before wiring
   them together.
6. **Naming the constituents enables later substitution.** Replacing
   KITE with BALLOON updates the composed memory cheaply.
7. **The two wiring styles produce different recall behaviour.**
   Flat memory cannot share structure with related memories; composed
   memory shares structure by construction.

## What the implementation already absorbs

### 1 — K-lines may cite other K-lines

K-line `useful_context.prior_klines: [ ... ]` records prior K-lines
that contributed to the present settlement
([`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
[`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)).
The link from a new K-line to its predecessors exists.

### 2 — Typed relational links

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
gives every durable record `linked: [ {kind, target} ]` with kinds
including `derived_from`, `cites`, `reinforces`, `analogous_to`. The
graph substrate for composed memories exists.

### 3 — Relational index built on read

A lazy index over relational links is described in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md).
Traversal from a K-line to its cited predecessors is in principle
supported.

### 4 — Append-only supersession

Corrections happen by new entries with `supersedes:` rather than
in-place edits. This preserves the composed memory's history when
constituents are revised.

## What the implementation does not yet take into account

### A — `prior_klines` is citation, not composition

Reactivating a parent K-line does not transitively reactivate its
sub-K-lines. `klines.ts` selects K-lines by cue similarity and
boosts their `activation_snapshot` — it does not walk
`prior_klines` to also activate the children. The figure's
right-hand panel — "K connects to three K-lines, which connect to
the leaves" — is not how reactivation flows.

### B — No `composed_of` field

There is no schema slot that says "this K-line *is* the conjunction
of these K-lines." `prior_klines` records that earlier K-lines
*contributed*, not that the present K-line is built from them as
structural parts. The composed form of Figure 8-7 has no native
expression.

### C — Substitution is rewrite, not edit

Swapping KITE for BALLOON in a composed memory has no primitive in
the plan. The append-only `supersedes` convention rewrites the
whole K-line. The figure's economy — change one constituent, the
whole memory updates — is not available.

### D — Decay treats K-lines independently

`reuse_count` and `decay_score` are per-K-line
([`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)).
A composed K-line does not keep its children alive by reference,
and a heavily reused child does not lift its parents. The
inheritance the figure describes does not propagate through decay.

### E — Recognition of constituents is not modelled

Figure 8-7 assumes the composer first *recognised* JACK, FLY, and
KITE — a cognitive act prior to wiring. The plan's `archivist` writes
K-lines from the final settlement; it does not run a recognition pass
that identifies which prior K-lines best summarise the current
episode.

### F — No "share structure with related memories" benefit

The flat-vs-composed information cost is invisible to the plan. Two
K-lines that share the same JACK constituent do not actually share
storage or activation in the flat form; nor does composition share
it in the current schema (since `prior_klines` is a list of ids, not
a structural link the runtime exploits).

## Implication for the plan (no changes proposed here)

Figure 8-7 turns memory from a list of episodes into a *graph* of
reusable parts. The plan has the graph substrate (`linked:`,
`prior_klines`) but treats K-line reactivation as a flat selection
over independent records. Closing the gaps would touch the K-line
schema and loader in
[`06-frames-polynemes-klines.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/06-frames-polynemes-klines.md),
the write rules and decay treatment in
[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
and the archivist's role in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
