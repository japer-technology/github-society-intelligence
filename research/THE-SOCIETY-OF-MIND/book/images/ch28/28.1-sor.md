# Figure 28-1 — Implementation analysis (SOR / FORGEJO-SOCIETY-IMPLEMENTATION)

**Source figure:** [28-1.md](28-1.md) — *Epigraph "Flux" by Theodore
Melnechuk.*
**Analysis target:** [`FORGEJO-SOCIETY-IMPLEMENTATION/`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/)
(plan-of-record for `.forgejo-society/` and
`.forgejo/workflows/forgejo-society.yaml`).

This file inventories the ideas carried by Figure 28-1 and checks each
one against the implementation plan. The aim is to mark, plainly, what
the plan already absorbs and what it does not yet name.

---

## The ideas Figure 28-1 actually carries

The figure is an epigraph, but it is positioned to do load-bearing
work for the chapter on the *myth of mental energy*. Pulled from the
poem and from its placement at the head of §28:

1. **Every event is singular.** No occurrence recurs. A system that
   pretends two runs are "the same" is reading over a difference.
2. **Apparent stability rides on continuous change.** "Hard things
   slowly wear away" — even the substrate one assumes is fixed is
   drifting.
3. **A single name labels a churning aggregate.** The "twinkling sea"
   is named once though it is "ten billion waves." The name is a
   convenience for the namer, not a property of the thing.
4. **Identity is a bookkeeping convention.** What persists across the
   flux is the *label*, kept by something outside the waves, not the
   waves themselves.
5. **Stable currencies arise to regulate transactions.** The chapter's
   own move: agencies invent quantities (energy, effort, priority)
   precisely *because* they need a tradable unit between parts that
   are otherwise incommensurable.
6. **Such currencies are useful fictions.** They are not measurements
   of an underlying substance; they are accounting tokens whose value
   is the agreement to honour them.
7. **Twenty-seven thousand days.** A finite, countable horizon. The
   self is not eternal scaffolding; it is a long-but-bounded ledger.
8. **The labeller is distinct from the labelled.** Something has to
   *keep* the single name across the waves. That something is itself
   part of the society, not above it.

These eight items are what the rest of this file tests against the
implementation plan.

## What the implementation already absorbs

### 1 — Each run is a distinct, addressed event

[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)
and [`09-handoff-and-signal-schemas.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/09-handoff-and-signal-schemas.md)
treat every workflow execution as a uniquely identified event with
its own `event_id`, `run_id`, and settlement. Two runs over the same
inputs are not collapsed into one; the figure's "nothing happens
twice" is honoured at the event level.

### 2 — Stable names over changing content

[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
gives every unit a stable id (`agency.code.patch-imaginer`, etc.) that
persists across many runs, even while the unit's prompt body,
budgets, and behaviour evolve. The id is the "single name"; the
prompt and the run are the "twinkling sea." This is exactly the
labelling pattern the poem describes.

### 3 — Memory as ledger, not as substance

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
treats memory as appended records (K-lines, settlements, signals)
under `memory/`, addressable by id, with no claim that the records
*are* the agent. The agent is whatever the active manifest says it is
now; memory is the wake.

### 4 — Settlements as the unit of "what happened"

The settlement schema in
[`03-runtime-pipeline.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/03-runtime-pipeline.md)
produces one auditable record per run. That record is the operational
form of "this particular wave, named." It is the closest the plan
comes to the figure's bookkeeping move: a discrete artefact that
*stands in* for a flux of internal activity.

### 5 — Budgets as a tradable currency

`budget.max_tool_calls` and `budget.max_wall_clock_s` in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md)
function as a primitive shared currency between agencies. They are
the plan's nearest analogue to Minsky's argument that mental
quantities exist because parts need to bargain. The currency is small
and physical (calls, seconds), but it exists.

## What the implementation does not yet take into account

These are the parts of Figure 28-1 that the plan currently leaves
implicit, partial, or simply absent. None of them are urgent; all of
them are real.

### A — No name for "the labeller"

The poem turns on a quiet asymmetry: someone *keeps* the single name
across the waves. In the plan, ids are kept by the filesystem and by
git history; no agency owns the act of *maintaining identity over
time*. There is no `agency.identity.namekeeper`, no censor that
refuses to silently rebind an id, and no critic whose job is to
notice that "the unit called `patch-imaginer` today is not the unit
that ran under that name last month."

### B — Drift across runs is not first-class

[`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md)
records what each run produced, but the plan has no field that
records *how much the unit itself has changed* between runs — no
manifest-diff summary, no rolling "behavioural drift" measure. "Hard
things slowly wear away" is, in the plan, only visible to a human
reading `git log` on a manifest file.

### C — No internal currency beyond budget

Minsky's argument is broader than wall-clock and call-count. The
chapter is about *priority*, *effort*, *attention*, *salience* — the
internal quantities agencies invent to settle disputes. The plan does
not yet name any of these. `critic.scope`, `critic.evidence`, and the
censor stack reject or pass actions; they do not bid against each
other on a shared scale, and there is no settlement field that
records what the winning bid was.

### D — Settlement is per-run, not per-identity

The settlement schema closes a run. It does not roll a unit's history
forward into a single, name-keyed ledger of "what this agency has
been over its twenty-seven thousand days." Memory is per-event;
identity-over-time is left to the implicit aggregate of git history,
which is not addressable by the runtime.

### E — "Same name" is not enforced as an honest claim

Nothing in the plan currently asks: *is this still the same unit?*
A manifest can be rewritten end-to-end while keeping its id, and no
critic will object. The figure's quiet warning — that calling a sea
by one name is a choice — has no operational analogue. There is no
threshold past which `agency.code.patch-imaginer` should be retired
in favour of `agency.code.patch-imaginer.v2`.

### F — The finite horizon is not represented

"Twenty-seven thousand days" is a bounded ledger. The plan has no
notion of a unit's lifetime, retirement, or sunset. Agencies persist
until removed by hand. There is no `lifecycle.retire_after:` field,
no `agency.meta-admin.retirement-broker`, and no scheduled review.
Whatever drifts, drifts indefinitely.

### G — Currencies as fictions are not labelled as such

If budgets are accounting tokens (and the chapter argues they must
be), the plan does not say so. `budget.max_tool_calls` reads as a
hard physical limit, not as a negotiated quantity. There is no
documentation surface that frames internal quantities as *agreed
fictions*, and so nothing prevents later code from treating them as
measurements of an underlying substance — exactly the move Minsky
calls the *myth of mental energy*.

### H — No mechanism for honest re-identification

If a unit changes enough that it is no longer the same unit, the
plan offers no path for the system itself to say so. Re-identification
would have to be done by a human editing a manifest. There is no
`event.identity.rebound` event type, no settlement that records
"identity continuity assessed: held / broken," and no audit trail
specific to the question the poem asks.

## Implication for the plan (no changes proposed here)

Figure 28-1 is the figure that warns against confusing *the name* with
*the thing*. The implementation plan keeps stable names well, and
records each run honestly, but it does not yet have any unit whose
job is to *maintain identity over time*, nor any vocabulary for
treating internal currencies as agreed fictions rather than
measurements.

The gaps above are recorded here as analysis, not as a change
request. Any move to close them — for example, an
`agency.identity.namekeeper`, a `lifecycle.retire_after:` field, an
internal-currency registry, or a settlement field for
identity-continuity assessment — would touch the manifest schema in
[`05-agencies-critics-censors.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/05-agencies-critics-censors.md),
the memory model in [`08-state-and-memory.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/08-state-and-memory.md),
the policy surface in [`07-policies-and-safety.md`](../../../../../FORGEJO-SOCIETY-IMPLEMENTATION/07-policies-and-safety.md),
and the identity scopes in `THE-SOCIETY-OF-REPO/02-protocols/01-identity.md`,
and so falls under the "stop and ask" rules in
[`AGENTS.md`](../../../../../AGENTS.md) §12 and
[`CLAUDE.md`](../../../../../CLAUDE.md).
